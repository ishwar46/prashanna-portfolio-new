  <files>src/lib/email.ts, src/app/actions/contact.ts</files>
  <action>
**1. Create `src/lib/email.ts`:**
- Import nodemailer.
- Create a transporter using Gmail SMTP:
  ```
  host: process.env.SMTP_HOST || "smtp.gmail.com"
  port: 587
  secure: false
  auth: { user: process.env.SMTP_USER, pass: process.env.SMTP_PASS }
  ```
- Export `sendLeadEmail(data)` function that sends an HTML email:
  - From: `process.env.SMTP_USER`
  - To: `"prashanna@loanfactory.com"`
  - Subject: `"New Lead from Portfolio: ${data.name} - ${data.loanType || 'General Inquiry'}"`
  - HTML body: clean, professional template showing visitor name, phone, email (if provided), selected loan type, and message. Use inline CSS (not Tailwind). Include Prashanna's branding. Keep it simple and readable.
- Do NOT send any auto-reply to the visitor (per user decision).

**2. Create `src/app/actions/contact.ts`:**
- Mark as `"use server"`.
- Import `z` from `zod` and `sendLeadEmail` from `@/lib/email`.
- Define Zod schema:
  - `name`: string, min 1 ("Name is required")
  - `phone`: string, min 7 ("Valid phone number is required")
  - `email`: string, email("Invalid email address").optional().or(z.literal("")) -- email is optional per user decision
  - `loanType`: string, optional
  - `message`: string, optional
- Define `ContactState` type: `{ success?: boolean; message?: string; errors?: Record<string, string[]> }`
- Export `submitContact(prevState: ContactState, formData: FormData): Promise<ContactState>`:
  1. Honeypot check: if `formData.get("website")` has a value, return `{ success: true, message: "Thank you!" }` (silently succeed to not reveal trap).
  2. Parse and validate with Zod safeParse.
  3. If validation fails, return `{ success: false, errors: validatedFields.error.flatten().fieldErrors }`.
  4. Try `sendLeadEmail(validatedFields.data)`. On success: return `{ success: true, message: "Thank you! Prashanna will be in touch within 24 hours." }`.
  5. On catch: return `{ success: false, message: "Something went wrong. Please try again or call (571) 222-5555 directly." }`.

**Important:** Do NOT set `runtime = "edge"` anywhere -- Nodemailer requires Node.js runtime (the default).
