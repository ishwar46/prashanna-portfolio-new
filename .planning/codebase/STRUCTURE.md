## Directory Purposes

**public/:**
- Purpose: Static assets served at root URL
- Contains: SVG files (next.svg, vercel.svg, globe.svg, file.svg, window.svg)
- Key files: None; only static assets

**src/:**
- Purpose: Application source code
- Contains: Page components, layouts, reusable components, utilities
- Key files: `app/layout.tsx` (root), `app/page.tsx` (home), `components/ui/button.tsx` (reusable)

**src/app/:**
- Purpose: App Router pages and layouts (file-based routing)
- Contains: `.tsx` files that map to routes and layouts
- Key files:
  - `layout.tsx`: Root layout wrapping all pages
  - `page.tsx`: Home page (`/` route)
  - `globals.css`: Global styles, design tokens, Tailwind theme

**src/components/:**
- Purpose: Reusable React components
- Contains: UI primitives and custom components
- Key files: `ui/button.tsx` (CVA-based button component)

**src/components/ui/:**
- Purpose: UI primitives (from shadcn/ui)
- Contains: Single-responsibility components (Button, etc.)
- Key files: `button.tsx` with variant system

**src/lib/:**
- Purpose: Utility functions and helpers
- Contains: Helper functions for common tasks
- Key files: `utils.ts` with `cn()` class merging utility

**.planning/codebase/:**
- Purpose: Analysis documents for future work
- Contains: ARCHITECTURE.md, STRUCTURE.md, CONVENTIONS.md, etc.
- Key files: None; generated by GSD mappers

**.next/:**
- Purpose: Next.js build output
- Generated: Yes (automatically on `next build`)
- Committed: No (.gitignored)

## Key File Locations

**Entry Points:**
- `src/app/layout.tsx`: Root layout that wraps all pages; loads fonts and applies global styles
- `src/app/page.tsx`: Home page rendered at `/` route

**Configuration:**
- `package.json`: Dependencies and scripts
- `tsconfig.json`: TypeScript compiler options with path aliases (`@/*` → `src/*`)
- `next.config.ts`: Next.js settings (currently empty)
- `components.json`: shadcn/ui configuration for component generation
- `eslint.config.mjs`: ESLint rules (Next.js + TypeScript)
- `postcss.config.mjs`: PostCSS plugins (Tailwind)

**Core Logic:**
- `src/app/globals.css`: Design tokens (colors, spacing, radius) as CSS custom properties
- `src/lib/utils.ts`: `cn()` utility for class merging

**Components:**
- `src/components/ui/button.tsx`: Reusable Button with CVA variants

## Naming Conventions

**Files:**
- PascalCase for components: `Button.tsx`
- camelCase for utilities and helpers: `utils.ts`
- lowercase for directories: `ui/`, `components/`
- `layout.tsx` and `page.tsx` are reserved Next.js files for routing

**Directories:**
- lowercase with dashes for multi-word names (if any): future pattern
- By feature/type: `components/`, `lib/` organize by purpose

**Variables:**
- camelCase for functions and variables: `cn()`, `buttonVariants`
- PascalCase for components and types: `Button`, `ButtonProps`
- UPPERCASE for constants: `defaultVariants`

**Types:**
- PascalCase: `ButtonProps`, `VariantProps`
- Suffixed with `Props` for component prop types

## Where to Add New Code

**New Page:**
- Location: Create `.tsx` file in `src/app/` (or subdirectory for nested routes)
- Example: `src/app/about.tsx` creates `/about` route
- Use root layout automatically; add new layout.tsx in subdirectory if needed

**New Component:**
- UI Component: `src/components/ui/[ComponentName].tsx`
  - Export named component and variants (e.g., `buttonVariants`)
  - Use CVA for variant system if appropriate
  - Mark with `"use client"` if interactive
- Feature Component: `src/components/[Feature]/[ComponentName].tsx` (create as needed)

**New Utility:**
- Location: `src/lib/[utility-name].ts`
- Example: Create `src/lib/validators.ts` for form validation helpers

**Styling:**
- Global styles: Add to `src/app/globals.css` (CSS custom properties, base layers)
- Component styles: Use `className` with Tailwind classes (via `cn()` utility)
- Design tokens: Define in `globals.css` under `:root` and `.dark` selectors

## Special Directories

**.next/:**
- Purpose: Next.js build artifacts (compiled pages, static chunks)
- Generated: Yes (on `next build` or `next dev`)
- Committed: No (.gitignored)

**node_modules/:**
- Purpose: Installed npm dependencies
- Generated: Yes (on `pnpm install`)
- Committed: No (.gitignored)

## Configuration & Aliases

**Path Aliases (tsconfig.json):**
- `@/*` → `src/*` (allows `import { cn } from '@/lib/utils'`)

**shadcn/ui Aliases (components.json):**
- `@/components` → `src/components`
- `@/lib` → `src/lib`
- `@/hooks` → `src/hooks` (future)
- `@/utils` → `src/lib/utils`
